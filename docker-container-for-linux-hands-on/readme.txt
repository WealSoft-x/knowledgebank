■はじめに
まずdockerって何？っていう方は
Amazon Kindle Unlimitedで読める
「Docker&仮想サーバー完全入門　Webクリエイター＆エンジニアの作業がはかどる開発環境構築ガイド」
を読むことをオススメします。
とにかくLinuxのハンズオンをしたいんだ！って方は、
ChatGPTに「dockerFileやcompose.ymlを元にdockerコンテナを起動する方法を教えてください」
とか聞けば詳しく説明してくれるので、それでいいと思います。

簡単な使い方
powerShellで
docker compose up -d 
docker compose exec ubuntu bash
→これでdockerコンテナ内のbashでコマンドを入力していろいろ操作ができる状況になる

■このコンテナについて
このコンテナはAmazon Kindle Unlimitedで読める「Linuxで動かしながら学ぶTCP/IPネットワーク入門」
のハンズオンを手軽に行うために作成しました。
起動時に必要な物を自動でインストールし、ハンズオンで作成したネットワーク設定ファイルを自動でvolumeに永続化します。
なのでdocker compose downなどでコンテナを削除しても、再度作成されるときに以前のネットワーク設定ファイルが引き継がれます。

他にも永続化したい物がある場合は、言っていただければ自分がそのやり方をコンフルに書くかもしれません。
オススメとしては、ChatGPTに設定ファイルを食わせて質問しながら何をやっているか理解してアレンジする、です。

あるいはもっとスマートなやり方もあるかと思うので、
これをベースによりよい設定ファイルを造れたら、是非共有いただけると幸いです。

■dockerのvolumeに永続化されているネットワーク設定ファイルをホスト側に持ってきたい場合
1.
docker ps -a
（ubuntu_data を使用しているコンテナの コンテナID または 名前 をメモ）

2.
docker cp <コンテナIDまたは名前>:/data/netns ~/backup
